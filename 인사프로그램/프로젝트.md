`from sklearn.model_selection import train_test_split from sklearn.ensemble import RandomForestClassifier import pandas as pd  df = pd.read_csv("attendance_data.csv")  # 입력(X)과 출력(y) 나누기 X = df[["근무시간"]] y = df["지각여부"]  # 학습 데이터/테스트 데이터 나누기 X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)  # AI 모델 학습 model = RandomForestClassifier() model.fit(X_train, y_train)  # 정확도 확인 print("정확도:", model.score(X_test, y_test))`

## 🧩 1️⃣ 목적 정의 (무엇을 위한 인사 AI인가?)

먼저 AI의 **목적**을 명확히 해야 데이터 수집 방향이 정해집니다.

- **조직문화 AI** → 직원 만족도 분석, 커뮤니케이션 패턴 분석
    

👉 목표를 정해야 “무슨 데이터를 얼마나 수집해야 하는지” 판단할 수 있습니다.

## 🧩 1️⃣ 인사 프로그램의 전체 구조 개요

### 🔸 상위 구조 (3계층 구조 예시)

`┌────────────────────────────┐ │   Presentation Layer (UI) │ ← 웹/모바일 화면 ├────────────────────────────┤ │   Application Layer (Logic)│ ← 비즈니스 로직, AI, API ├────────────────────────────┤ │   Data Layer (Database)   │ ← PostgreSQL, MongoDB 등 └────────────────────────────┘`

---

## 🧱 2️⃣ 주요 모듈별 구성

대부분의 인사프로그램은 다음과 같은 **핵심 기능 모듈**로 나뉩니다 👇

| 모듈                                     | 주요 기능                      | 사용 데이터                               |
| -------------------------------------- | -------------------------- | ------------------------------------ |
| **① 인사기본관리 (HR Core)**                 | 인사기본정보, 부서, 직급, 근속정보, 인사이동 | `employee`, `department`, `position` |
| **② 근태관리 (Attendance)**                | 출퇴근, 휴가, 근무시간, 연차관리        | `attendance`, `leave`                |
| **③ 급여관리 (Payroll)**                   | 급여계산, 수당, 공제, 세금처리         | `salary`, `tax`, `benefit`           |
| **④ 채용관리 (Recruiting)**                | 공고등록, 지원자관리, 평가, 채용결정      | `applicant`, `recruit_process`       |
| **⑤ 성과평가 (Performance)**               | KPI, OKR, 평가등급, 보상연계       | `performance`, `goal`, `review`      |
| **⑥ 교육/경력개발 (Learning & Development)** | 교육이력, 자격증, 내부교육추천          | `training`, `certificate`            |
| **⑦ 조직관리 (Organization)**              | 조직도, 팀 구조, 직무정의            | `org_chart`, `role`                  |
| **⑧ AI / 분석모듈 (AI Analytics)**         | 이직 예측, 성과 예측, 만족도 분석       | 통합된 HR 데이터                           |
| **⑨ 관리자/권한관리 (Admin)**                 | 접근제어, 역할별 권한, 로그기록         | `user`, `role`, `permission`         |

## 🧬 3️⃣ 데이터베이스 구조 (핵심 테이블 예시)

`┌────────────────────┐ │ employees          │ │ ├ id (PK)          │ │ ├ name             │ │ ├ dept_id (FK)     │ │ ├ position_id (FK) │ │ ├ hire_date        │ │ ├ status           │ └────────────────────┘  ┌────────────────────┐ │ departments        │ │ ├ id (PK)          │ │ ├ name             │ │ ├ parent_id (FK)   │ └────────────────────┘  ┌────────────────────┐ │ attendance         │ │ ├ id (PK)          │ │ ├ employee_id (FK) │ │ ├ date             │ │ ├ check_in/out     │ └────────────────────┘  ┌────────────────────┐ │ payroll            │ │ ├ id (PK)          │ │ ├ employee_id (FK) │ │ ├ base_salary      │ │ ├ bonus            │ │ ├ tax              │ │ ├ pay_date         │ └────────────────────┘`

> 📘 참고: 실제 설계 시에는 `foreign key`, `index`, `audit_log` 등을 세밀히 설정합니다.

## 🔒 7️⃣ 보안 및 권한 구조

- **RBAC(Role-Based Access Control)**  
    → 관리자(Admin), 인사담당자(HR), 직원(Employee), 경영진(Manager)
    
- **감사 로그 (Audit Log)**  
    → 누가 언제 어떤 데이터를 수정했는지 추적
    
- **데이터 마스킹 / 익명화**  
    → AI 학습용 데이터는 개인정보 제거 후 사용
