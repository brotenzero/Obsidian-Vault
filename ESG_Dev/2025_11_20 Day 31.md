### 2025년 지금 기준으로 추천하는 순위

1. **async + await** ← 무조건 이걸 제일 먼저 배워! (99%의 경우 이걸로 충분)
2. Promise ← await은 결국 Promise라서 알아야 함
3. Callback ← 옛날 라이브러리나 Node.js 기본 API에서 가끔 보임
4. Top-level await ← Next.js, Vite 같은 최신 프로젝트에서 점점 많이 씀

### none / null / undefined 초보자도 3초만에 끝나는 비교표! ✨

| 언어         | 값 이름      | 뜻 (한마디로)        | 누가 넣었나?         | 예시 코드                                     | 언제 쓰나?                 |
| ---------- | --------- | --------------- | --------------- | ----------------------------------------- | ---------------------- |
| **Python** | None      | “일부러 아무것도 없다”   | 프로그래머가 직접 넣음    | name = None                               | 함수에서 값 없을 때 리턴, 초기값    |
| **JS/TS**  | null      | “일부러 아무것도 없다”   | 프로그래머가 직접 넣음    | let name = null;                          | 객체가 없거나, 초기값, API에서 없음 |
| **JS/TS**  | undefined | “아직 아무 값도 안 줬다” | 자바스크립트가 자동으로 넣음 | let name; console.log(name); // undefined | 변수 선언만 했을 때, 함수 리턴 안 함 |

|?|옵셔널 체이닝 (Optional Chaining)|“이게 있으면 접근하고, 없으면 바로 undefined 줘”|user?.name?.age|객체 깊숙이 들어갈 때 null/undefined 때문에 에러 나는 거 방지!|
|?.|(위와 같은데 점 붙은 버전)|함수나 배열도 안전하게 호출|user?.getName?.() list?.[0]|제일 자주 씀! 2025년엔 거의 모든 코드에 들어감|
