public class RockPaperScissorsSwitch {

    public static void main(String[] args) {

        Scanner sc = new Scanner(System.in);

        System.out.println("가위바위보 게임입니다. 0(가위), 1(바위), 2(보) 중에서 입력하세요");

        System.out.print("사용자 입력 >> ");

        int user = sc.nextInt();

        int computer = (int)(Math.random() * 3);

  

        System.out.println("컴퓨터: " + computer + ", 사용자: " + user);

  

        switch (user - computer) {

            case 2: case -1:

                System.out.println("컴퓨터가 이겼습니다.");

                break;

            case 1: case -2:

                System.out.println("사용자가 이겼습니다.");

                break;

            default:

                System.out.println("비겼습니다.");

                break;

        }

        sc.close();

    }

}

## 어노테이션(Annotation)

- **주석, 메타데이터**라는 의미로, Java와 같은 프로그래밍 언어에서 코드에 특별한 정보를 부가하거나 컴파일러와 프레임워크가 해석할 수 있게 해주는 문법적 표식입니다.[](https://velog.io/@rara_kim/Spring-%EC%96%B4%EB%85%B8%ED%85%8C%EC%9D%B4%EC%85%98Annotation)​
    
- 예를 들어, 자바에서 `@Override`, `@RestController` 등은 컴파일러나 프레임워크가 특정 처리나 검사를 할 수 있게 힌트를 주는 역할입니다.
    
- 사용 예시:
    
    java
    
    `@Override public String toString() { ... }`
    
- 데이터 어노테이션은 데이터 셋에 라벨﻿을 달아 추가정보를 부여하는 것도 의미합니다.[](https://blog.naver.com/saltlux1979/222315111818)​

## 노테이션(Notation)

- **표기법, 기호 체계**라는 의미로, 특정한 규칙과 약속에 따라 어떤 대상을 표현하거나 나타내는 방법을 의미합니다.
    
- 수학에서의 수식 표기법(중위 표기법, 전위 표기법, 후위 표기법), UML 다이어그램, 음악 기보법 등 다양한 분야에서 사용됩니다.[](https://translate.google.com/translate?u=https%3A%2F%2Fwww.quora.com%2FWhat-is-the-difference-between-annotation-and-notation&hl=ko&sl=en&tl=ko&client=srp)​
    
- 예시:
    
    - 수학에서 a+ba+b처럼 쓰는 중위 표기법
        
    - 프로그래밍에서 JSON은 데이터 구조를 표기하는 "노테이션"입니다.

## POJO의 특징
plain old java object


- **순수 자바 객체**: 자바 언어의 기본 문법만으로 작성된 클래스입니다.
    
- **특별한 상속·구현·어노테이션이 필요 없음**: javax.persistence.Entity, Serializable, 특정 인터페이스(예: EJB 등)에 의존하지 않습니다.
    
    - 즉, `extends`나 `implements` 없이 필드, 생성자, getter/setter 등만 가지고 있으면 POJO입니다.[](https://siyoon210.tistory.com/120)​
        
- **프레임워크 독립**: 어떤 프레임워크(Spring, Hibernate 등)와도 전혀 결합되어 있지 않아서, **재사용**과 **테스트**가 매우 쉽습니다.[](https://xogns93.tistory.com/95)​
    
- **객체지향적 설계**에 충실하며, 필요한 경우 프레임워크가 이를 감싸서 확장하거나 관리하게 만듭니다.

**@Data**

- 클래스의 모든 필드에 대해 `@Getter`, `@Setter`, `@ToString`, `@EqualsAndHashCode`, `@RequiredArgsConstructor` 기능을 한 번에 자동 생성해주는 Lombok의 종합 어노테이션입니다.
    
- 즉, 이 어노테이션 하나만 붙이면, 필드 접근자/설정자, toString(), equals(), hashCode() 등 모든 보일러플레이트 코드가 자동 작성됩니다.[](https://zi-c.tistory.com/19)​
    
- 주로 **DTO·VO 등 데이터 모델 객체**에서 사용하며, 코드가 매우 간결해지는 장점이 있습니다.
    

**@AllArgsConstructor**

- 클래스의 **모든 필드를 매개변수로 받는 생성자**를 자동으로 만듭니다.
    
- 예를 들어, `new User(id, name, email)`처럼 객체를 한번에 모든 필드 값으로 생성할 때 사용됩니다.[](https://velog.io/@sunil1369/Java-Lombok%EC%9D%98-Data-%EC%96%B4%EB%85%B8%ED%85%8C%EC%9D%B4%EC%85%98)​
    

**@NoArgsConstructor**

- **파라미터가 없는(빈) 기본 생성자**를 자동으로 만듭니다.
    
- JPA, Jackson 등 프레임워크와 연동할 때 필수로 요구되는 경우가 많아 자주 함께 쓰입니다.[](https://velog.io/@sunil1369/Java-Lombok%EC%9D%98-Data-%EC%96%B4%EB%85%B8%ED%85%8C%EC%9D%B4%EC%85%98)​
