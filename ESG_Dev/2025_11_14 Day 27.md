# 1️⃣ 컴포넌트 스캔(Component Scan) 정의

> **컴포넌트 스캔이란, 스프링이 특정 패키지를 탐색하여 `@Component`, `@Service`, `@Repository`, `@Controller` 등의 어노테이션이 붙은 클래스를 찾아서 자동으로 스프링 컨테이너에 빈(bean)으로 등록하는 과정**을 말합니다.

즉, **“내가 만든 클래스를 일일이 수동 등록하지 않아도, 스프링이 알아서 찾아서 관리하게 해주는 기능”**입니다.

# 1️⃣ 스프링 컨텍스트(ApplicationContext)

- **정의:** 스프링에서 **빈(Bean) 객체를 관리하는 컨테이너**
    
- **역할:**
    
    - 의존성 주입(DI)
        
    - 빈의 라이프사이클 관리(초기화, 소멸)
        
    - AOP, 이벤트, 환경 설정 관리
        
- **관리 범위:** 애플리케이션 전반 (싱글톤 빈 등)
    
- **생명주기:** 애플리케이션 시작 시 생성 → 종료 시 소멸
    

예시: `@Service`, `@Repository`, `@Controller` 등의 빈, `DataSource`, `EntityManagerFactory`



# 2️⃣ JPA 퍼시스턴스 컨텍스트(Persistence Context)

- **정의:** JPA에서 **엔티티 객체를 관리하고 1차 캐시에 저장하는 환경**
    
- **역할:**
    
    - Entity 객체의 상태를 관리 (Managed 상태)
        
    - 변경 감지(Dirty Checking)
        
    - DB와의 동기화 (`flush`)
        
- **관리 범위:** 트랜잭션 단위
    
- **생명주기:**
    
    - `EntityManager`가 생성될 때 퍼시스턴스 컨텍스트가 만들어지고
        
    - 트랜잭션 종료 시 소멸(또는 명시적 종료)
