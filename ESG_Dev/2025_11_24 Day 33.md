### 무상태(Stateless) 프로그래밍이란?

쉽게 말해서 **“한 번 요청이 끝나면 그 요청에 대한 모든 기억을 완전히 잊어버리는 방식”**입니다.

| 구분      | 무상태 (Stateless)                                  | 유상태 (Stateful)              |
| ------- | ------------------------------------------------ | --------------------------- |
| 기억      | 기억 안 함                                           | 기억함                         |
| 요청 하나하나 | 완전히 독립적                                          | 이전 요청을 기억하고 있어야 다음 요청 처리 가능 |
| 대표 예시   | HTTP 웹 요청, REST API, 함수형 프로그래밍                   | 로그인 세션, 쇼핑카트, 웹소켓, DB 트랜잭션  |
| 장점      | - 확장하기 쉬움 (서버 수백 대로 마음대로 늘림) - 장애 복구 쉬움 - 예측 가능함 | - 개발하기 편함 (이전 상태를 기억하니까)    |
| 단점      | - 매번 모든 데이터를 요청에 넣어야 함                           | - 서버가 터지면 상태 날아감 - 스케일링 어려움 |

# **블로킹(Blocking) 정의**

**블로킹(Blocking)** 이란 **스레드가 어떤 작업이 끝날 때까지 기다리는 상태**를 말합니다.

- 작업이 완료될 때까지 **스레드가 다른 일을 못 함**
    
- 스레드가 “멈춰 있음” → CPU나 리소스를 효율적으로 못 씀
    

즉, **한 스레드당 한 작업**을 처리할 때, 그 작업이 완료될 때까지 스레드가 점유된 상태라고 생각하면 됩니다.
