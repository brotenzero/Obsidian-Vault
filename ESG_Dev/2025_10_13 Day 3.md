## 1. 스타트업 / 제품 개발에서의 MVP

“MVP”는 **Minimum Viable Product (최소 기능 제품)**의 약어야. “가장 단순하면서도 고객에게 가치를 줄 수 있는 형태의 제품”이라는 뜻이지. 이 개념은 린 스타트업(Lean Startup)의 핵심 아이디어 중 하나야.

localhost.8080
`localhost:8080`은 개발자가 자신의 컴퓨터에서 웹 서버를 실행하고 테스트할 때 사용하는 주소입니다. 이 주소를 통해 로컬 환경에서 웹 애플리케이션을 개발하고 확인할 수 있습니다.
Apache Tomcat 서버를 로컬에서 실행할 때도 기본 포트로 `8080`을 사용합니다. 예를 들어, `http://localhost:8080/manager`에 접속하면 Tomcat의 관리자 페이지에 접근할 수 있습니다.

## ✅ 소스(Source)

**정의**: 소스는 소프트웨어의 **원본 코드**를 의미합니다. 이는 사람이 읽을 수 있는 형태로 작성된 프로그램 코드로, 주로 텍스트 파일로 저장됩니다.



## ✅ 리소스(Resource)

**정의**: 리소스는 소프트웨어가 실행되는 동안 필요로 하는 **외부 파일**이나 **데이터**를 의미합니다. 이는 프로그램의 실행에 직접적인 영향을 미치지 않지만, 프로그램의 기능을 보완하거나 지원하는 역할을 합니다.

## 1. MVC란 무엇인가?

- **MVC (Model‑View‑Controller)**는 애플리케이션을 세 부분으로 분리해서 설계하는 패턴이야. 각 부분이 서로 다른 책임을 지면서 서로 협력하는 구조지. [위키백과+2GeeksforGeeks+2](https://en.wikipedia.org/wiki/Model%E2%80%93view%E2%80%93controller?utm_source=chatgpt.com)
    
- 이렇게 분리하면 코드가 깔끔해지고, 수정·확장이 쉬워지며, 여러 개발자가 동시에 작업하기 수월해져.

디자인패턴

| 구성 요소          | 책임 / 역할                                                                | 예시                                            |
| -------------- | ---------------------------------------------------------------------- | --------------------------------------------- |
| **Model**      | 데이터와 비즈니스 로직을 관리해. DB 조작, 상태 저장, 유효성 검사 등이 여기서 일어나지.                   | 사용자 정보, 게시글 데이터, 계산 로직 등                      |
| **View**       | 사용자에게 데이터를 보여주고, 사용자 입력을 화면에서 받는 역할이야. 즉 화면(UI) 부분.                    | HTML, 템플릿, 버튼, 그래프 등                          |
| **Controller** | 사용자의 요청을 받고, 그 요청을 처리하고, Model을 조작한 뒤, 적절한 View를 선택해서 보여줘. 중간 다리 역할이야. | URL 요청 → controller → model 호출 → 결과 → view 반환 |
|                |                                                                        |                                               |

### 📶 예시로 이해하기

예를 들어,  
너(사용자)가 어떤 웹사이트에 접속하고 싶다고 해보자.

보통은 이렇게 되죠:

> 너 🧍 → 웹사이트 🌐

그런데 프록시를 쓰면 이렇게 돼요:

> 너 🧍 → **프록시 서버** 🖥 → 웹사이트 🌐

즉, **프록시 서버가 대신 요청을 보내고, 결과를 너에게 전달**해주는 거예요.


### 🧩 프록시를 쓰는 이유

1. **익명성 보호**
    
    - 실제 내 IP 주소를 숨길 수 있어요.
        
    - 웹사이트는 프록시의 IP만 보게 됩니다.
        
2. **보안 강화**
    
    - 회사나 학교 네트워크에서 외부 접근을 제한하거나 감시할 때 사용돼요.
        
    - 위험한 사이트를 차단하기도 합니다.
        
3. **속도 향상 (캐싱)**
    
    - 자주 방문하는 웹페이지를 프록시가 저장해 두었다가 더 빠르게 보여줄 수 있어요.
        
4. **접근 우회**
    
    - 지역 제한(예: 한국에서는 안 되는 사이트)을 우회할 때 쓰기도 해요.


### 🔹 URL의 구조

| 부분                   | 예시                | 설명                                              |
| -------------------- | ----------------- | ----------------------------------------------- |
| **프로토콜**             | `https://`        | 어떤 방식으로 접속할지를 나타냄 (예: http, https, ftp 등)       |
| **도메인 이름(호스트)**      | `www.example.com` | 서버의 이름 또는 IP 주소                                 |
| **포트 번호**            | `:8080`           | 서버에서 사용하는 통신 포트 (보통 생략됨 — HTTP는 80, HTTPS는 443) |
| **경로(Path)**         | `/path/page.html` | 서버 안에서 자원의 위치                                   |
| **쿼리(Query String)** | `?search=apple`   | 추가 정보(변수)를 서버에 전달할 때 사용                         |
| **프래그먼트(Fragment)**  | `#section2`       | 문서 내 특정 위치(북마크 등)를 가리킴                          |
