###   
Access Token은 인증(Authentication)인가? 인가(Authorization)인가?

→ **둘 다 한다!!** 하지만 **주로 인가(Authorization)** 에 쓰인다!

초보자도 1분 만에 이해할 수 있게 정리해줄게요.

|질문|답변|비유 (클럽 다시 한 번!)|
|---|---|---|
|Authentication (인증)|"너 진짜 홍길동 맞아?"|신분증 검사 (너 누구냐?)|
|Authorization (인가)|"홍길동이면 VIP 룸 들어가도 돼"|출입증에 적힌 권한 확인|

### Access Token은 정확히 뭘 해?

|단계|무슨 일 일어나?|인증? 인가?|
|---|---|---|
|1. 로그인 (아이디+비번)|서버가 "너 진짜 홍길동 맞다!" 확인|→ 인증|
|2. Access Token 발급|서버가 "홍길동 너한테 이 출입증 줄게" 하고 토큰 줌|인증 끝난 후 발급|
|3. 내가 API 요청할 때|내가 요청마다 출입증(Access Token)을 들고 감||
|4. 서버가 토큰 검사|"이 출입증 진짜고, 아직 안 만료됐네 → 너는 게시물 읽기+쓰기 권한 있음"|→ 인가!!|

결론 **Access Token 자체는 “누구인지”(인증) 증명 + “무엇을 할 수 있는지”(인가) 권한을 동시에 담고 있다.** 하지만 실무에서는 거의 100% **“무엇을 할 수 있는지 확인하는 용도 = 인가(Authorization)”** 로 쓰인다!

### 제일 헷갈리는 Refresh Token까지 한 번에 정리

|토큰 종류|주로 하는 일|인증? 인가?|언제 쓰이나요?|
|---|---|---|---|
|Access Token|짧게 살아서 API 요청할 때 씀|주로 인가|매 요청마다 헤더에 넣어서 보냄|
|Refresh Token|Access Token 재발급용|인증|Access 만료됐을 때 새로 로그인 없이 재발급|
|ID Token (OIDC)|"너 누구야?" 정보만 담김|인증|구글/카카오 로그인 후 사용자 정보 보여줄 때|

### 한 줄 요약 (이거만 기억해!)

> **Access Token = 클럽 출입증** → 신분증 검사(인증)는 이미 끝났고, → 이제 이 출입증 보고 “너는 일반석? VIP석? 술 주문 가능?”을 판단 = **인가(Authorization)**
