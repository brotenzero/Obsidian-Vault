
- **클라이언트**: Flutter
- **서버**: Dart (shelf 프레임워크)  - Shelf 
		→ Dart Frog 2.0 (파일 기반 라우팅) 프레임워크 변경
- **데이터베이스**: PostgreSQL
- **AI 모델**: Python + PyTorch
- **벡터 검색**: FAISS
- **배포/운영**: Docker + Kubernetes + MLflow


# 1️⃣ 클라이언트 (앱/웹)

**추천:** **Flutter**

- 이유: iOS/Android 동시에 개발 가능 → 코드 재사용 높음
    
- 최신 UI 프레임워크, Google 지원, 웹/데스크톱 확장 가능
    
- 언어: Dart
    

---

# 2️⃣ 서버 (Backend)

**추천:** **Spring Boot + Kotlin**

- 이유: JVM 기반 최신 서버 언어 Kotlin과 결합 → 생산성, 안정성, 마이크로서비스 친화적
    
- REST/GraphQL API, 보안, DB 연동 표준
    
- 주력 프레임워크로 대기업/스타트업 모두 사용
    

---

# 3️⃣ 데이터베이스

**추천:** **PostgreSQL**

- 이유: 관계형 DB 표준, JSONB 지원, 확장성 높음
    
- Redis/Elasticsearch는 필요시 보조 기술로 추가 가능하지만 기본은 PostgreSQL
    

---

# 4️⃣ AI/ML 모델

**추천:** **Python + PyTorch**

- 이유: 가장 널리 사용되는 AI 프레임워크, 커뮤니티/라이브러리 풍부
    
- HuggingFace Transformers 포함 → NLP/추천/검색 등 현대적 AI 구현 가능
    

---

# 5️⃣ 벡터 검색/추천 (AI 모델 연계)

**추천:** **FAISS**

- 이유: Facebook 오픈소스 벡터 검색 라이브러리, 임베딩 기반 추천/검색 구현 표준
    
- Milvus나 다른 상용 벡터 DB보다 범용성 높음, 모델 임베딩과 직관적 연계 가능
    

---

# 6️⃣ 배포 & 운영

**추천:** **Docker + Kubernetes + MLflow**

- 이유:
    
    - **Docker**: 컨테이너화 → 환경 일관성, 배포 표준
        
    - **Kubernetes**: 자동 스케일링, 마이크로서비스 운영 표준
        
    - **MLflow**: AI 모델 실험/버전 관리, MLOps 표준
